Detailed Description

Functions and data definitions for implementation of Data Block tag.

INFO:

All data blocks have html tag properties, and html content part. TAGDataBlock structure holds all data block html tag properties and contents in order to print them later.
Content part of data block must have only one html tag which is

"<table></table>"


"<table>" tag can have:

"<thead></thead>"
"<tfoot></tfoot>"
"<tbody></tbody>"


"<TBODY>" tag must have only one "<TR>...</TR>" tag.

All contents of data block (head, foot and body) are converted to TAGGeneric object instances and stored as a linked list. Linked list can be iterated by starting with rootTAGGeneric variable. tailTAGGeneric variable is the last TAGGeneric instance which show the last tag "</tr>" or text in the "<TBODY>".

All html tags and texts inside "<TBODY>" are called DYNAMIC tag/content. Dynamic content is as repeated as row count of the select query result. TAGGeneric object has isDynamicContent property to indicate it is dynamic or not.

First dynamic tag instance address is stored in startTAG variable. Last dynamic tag instance address is stored in endTAG variable. startTAG and endTAG instances are assigned during *.frmx file parsing phase. Until the file is reloaded, these values remain same. So, there is no need to iterate to find them in rootTAGGeneric linked list.
