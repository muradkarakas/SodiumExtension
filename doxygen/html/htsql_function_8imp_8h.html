Detailed Description

Deals with calling HT/SQL functions.

STEPS FOR HOW TO RUN an HT/SQL FUNCTION:

PostObjectGeneric data type is defined as a storage for local variables and function arguments of the HTSQL function being running or to be called. Example usage can be found in ajax.c file

  • Step.1:
    Before a new function call is made, Caller is to store the current value of the rootPostObjectGeneric and currentPreObjectGeneric variables defined in MKSession object.

    Then MUST always set currentMKSession->rootPostObjectGeneric and currentMKSession->currentPostObjectGeneric to NULL since they are used in createANewFunctionCallStackStartPoint
    function.
    PostObjectGeneric *prevRootPostObjectGeneric = currentMKSession->rootPostObjectGeneric;
    PostObjectGeneric *prevCurrentPostObjectGeneric = currentMKSession->currentPostObjectGeneric;
    currentMKSession->rootPostObjectGeneric = NULL;
    currentMKSession->currentPostObjectGeneric = NULL;


  • Step.2:
    A new PostObjectGenetic object will be created and stored in the rootPostObjectGeneric variable defined in MKSession object by calling createANewFunctionCallStackStartPoint(..). The object created newly will be the start point for local variables and function arguments. This value should be assigned currentMKSession->currentPostObjectGeneric as well.

    currentMKSession->rootPostObjectGeneric = createANewFunctionCallStackStartPoint(currentMKSession, functionNameToBeCalled);


  • Step.3
    Add block name of the caller's block name as variable named "datablockname" to the stack by using following c function. (This not optional!)

    postFunctionAddArgumentsOrVariables(currentMKSession, "datablockname", Current Block Name );


  • Step.4
    Add HT/SQL function arguments and control block items's names/values to the stack by using following c function.

    postFunctionAddArgumentsOrVariables(..);


  • Step.5
    Run the HTSQL function by calling the c function below. If return value is NULL, it means "functionNameToBeCalled" HT/SQL function was not found and not executed. Otherwise, it is found and executed. In that case return value points the function pre generic object structure. Do not destroy return value.

    PreObjectFunction *functionExecuted = runPostFunction(currentMKSession, functionNameToBeCalled, true or false);


  • Step.6.
    After execution the HTSQL function ends, the "function return value" will be in the object instance (functionExecuted->preObjectFunctionReturnValue->varcharValue) which was created in Step.2.

    functionExecuted->preObjectFunctionReturnValue->varcharValue


  • Step.7:
    After execution of the HT/SQL function ends, caller is responsible of destroying all the linked list. If this is ignored, the start point of the linked list will be lost and memory leak is inevitable.

    cleanUpFunctionStack(currentMKSession, page);


  • Step.8:
    After execution of the HTSQL function and destroying linked list, as a last step, caller should then restore the orjinal value of the currentMKSession->rootPostObjectGeneric and currentMKSession->currentPostObjectGeneric variable.

    currentMKSession->rootPostObjectGeneric = prevRootPostObjectGeneric;
    currentMKSession->currentPostObjectGeneric = prevCurrentPostObjectGeneric;


    Initially, rootPostObjectGeneric variable is assigned NULL. Procedure can call another procedure or itself again and again. IF all steps above is followed properly, rootPostObjectGeneric variable should be NULL since all callers are responsible to store old value back.

Finds HT/SQL function, then execute it.

Parameters
currentMKSessionMKSession*
htsqlFunctionNameconst char*
HT/SQL function name. Should not include return type, parameter list of the function to be called.
ignoreNotFoundbool
TRUE: "Function not found" error message does not be produced if it is not exists. This parameter is required for some optional HT/SQL functions (etc. "post_query" function of Data Block). If it is not exists, this does not mean an error. It is optional.
Returns
PreObjectFunction* PreObjectFunction pointer. It is the pointer to a HT/SQL function pre object instance which was executed. If function is optional and not found returns NULL.

Return value of the HT/SQL function will be stored in the properties of the preObjectGeneric instance. We are not creating a new instance. Instead just referencing to that instance
